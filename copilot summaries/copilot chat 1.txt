Current Functional Status
Admin / Security

Admin access uses a cookie-based unlock flow.
The unlock flow was fixed for Vercel by using a hard redirect after successful unlock instead of only router.push(...).
middleware.ts currently protects /admin/*.
There is still a Next.js warning that middleware is deprecated in favor of proxy, but it is only a warning right now.

Booking Detail / Edit / Delete

Public booking detail page:

used to show Edit/Delete publicly
latest direction is:

Edit/Delete should only show if admin is already unlocked
otherwise public users only see view details + calendar/admin navigation




Booking edit page:

should be protected by admin cookie check
if not unlocked, redirect to:

/admin/unlock?next=/bookings/[id]/edit




Booking PATCH/DELETE API:

should be protected by the same admin cookie check
unauthorized users should get 401 Unauthorized
