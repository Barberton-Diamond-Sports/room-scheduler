Important auth bug that was solved
The “have to log in every time I change admin pages” issue was eventually traced to the admin dashboard page.
Root cause:
The dashboard had a Logout control using Link to:
TypeScript/api/admin/logoutShow more lines
That caused the cookie to disappear when visiting /admin.
Fix:
On src/app/admin/page.tsx, the Logout control must be a plain anchor:
TypeScript/api/admin/logoutLogout</a>Show more lines
NOT:
TypeScript/api/admin/logoutLogout</Link>Show more lines
That was the fix that finally stabilized login persistence.
